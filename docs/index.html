Inter-Planetary File Rust System

Контент-адресуемая P2P-система
с ML-интеллектом

Распределённая файловая система на Rust, где каждый блок — сам себе адрес (CID = hash данных). Поверх классического IPFS добавлены семантический поиск, нейро-символический вывод и гео-распределённое исполнение графов между пирами.

RoadMap · Phase 5

Гео-распределённый вывод

Свежая веха: вычислительный граф разбивается на стадии и исполняется на разных пирах, а активации стримятся между ними по libp2p. Реализовано через инверсию зависимостей — без цикла network ↔ tensorlogic.

/ipfrs/activation/1.0.0libp2p request-response протокол — стриминг активаций между пирами (паттерн, как у block-fetch / semsearch).
TlExecutor — сменный бэкендConformist-трейт: NumExecutor над NumTensor, обобщённый einsum (matmul/matvec/dot/outer/batched) и N-D reduce. Тяжёлый движок можно подставить позже.
Планировщик пайплайнаАвто-разбиение графа на самодостаточные стадии по размещению node→peer, топологический порядок, contiguous-сплит для pipeline-параллелизма.
Гео-роутингВыбор пира по измеренному RTT и региону, hedge-запросы (plan_routing), выкачка по /ipfrs/blockfetch/1.0.0 с проверкой cid == hash(data).
Численные kernel'ыСтабильные f32-ядра: softmax (log-sum-exp), layer-norm, RMS-norm, GELU, SiLU — ACL-порт, считают по-настоящему.

DoD — интеграционный тест

2-stage граф · 2 живые libp2p-ноды

  УЗЕЛ A (оркестратор)        УЗЕЛ B (executor)
  ─────────────────        ─────────────────
  initial{x,b1,b2}
     │  stage 1 ──────▶ execute_stage
     │                       h = relu(x+b1)
     │  ◀────── activation h  (по сети)
  stage 2 (локально)
  y = h * b2
     │
     ▼
  y == [30, 0]  ✓  (== монолитный граф)
