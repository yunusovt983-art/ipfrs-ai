# IPFRS · S3 Console

S3-style веб-консоль для IPFRS — контент-адресуемое объектное хранилище поверх IPFS.
React + TypeScript + Vite, без внешних UI-зависимостей.

<p align="center"><em>Бакеты · виртуальные папки · загрузка/скачивание · CID · превью · demo/live режимы</em></p>

## Идея

У IPFS нет нативной модели «бакет / объект» — всё адресуется по CID. Консоль
накладывает S3-словарь поверх этого:

- **Бакет** — именованный манифест (список объектов), хранится в `localStorage`.
- **Объект** — пара `ключ → CID` + метаданные (размер, тип, дата, pin).
- **Папки** — виртуальные, выводятся из `/` в ключах (как в S3).

## Режимы

| Режим | Что делает | Где живут байты |
|-------|-----------|-----------------|
| **Демо** (по умолчанию) | Полностью офлайн. Загрузка считает детерминированный псевдо-CID (SHA-256 → base32, вид `bafybei…`). | in-memory + метаданные в `localStorage` |
| **Live** | `POST /api/v0/add` на шлюз IPFRS → настоящий CID; скачивание через `/api/v0/cat`; опциональный `pin`. | IPFS-шлюз |

Переключение и адрес шлюза — в модале настроек (иконка ⚙ внизу слева). Клиент
работает с Kubo-совместимым API `ipfrs-interface` (`/api/v0/add`, `/cat`,
`/version`, `/id`, `/swarm/peers`, `/ipfs/:cid`).

> ⚠️ Для live-режима нужен запущенный шлюз и разрешённый CORS.

## Возможности

- Список бакетов с агрегатами, создание/удаление
- Навигация по папкам (хлебные крошки), список и сетка
- Загрузка перетаскиванием и кнопкой, прогресс, батчи
- Панель деталей: полный CID (копирование), тип, размер, класс хранения, pin,
  превью изображений
- Поиск по объектам, тёмная/светлая тема (persist)

## Разработка

```bash
npm install
npm run dev        # http://localhost:5273
npm run build      # tsc + vite → dist/
npm run typecheck
```

Сборка самодостаточна (`base: "./"`) и работает под любым под-путём —
в т.ч. как статика на GitHub Pages (`docs/console/`).
